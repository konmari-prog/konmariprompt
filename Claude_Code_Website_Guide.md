# Claude Code × GitHub で Web サイトを作る方法

## はじめに

このガイドでは、Claude Code（AIアシスタント）とGitHub Pagesを使って、プロンプト集サイトを作成した手順を解説します。

---

## 必要なもの

- **GitHub アカウント** - 無料で作成可能
- **Claude Code** - AIアシスタントツール
- **画像素材** - サイトに使用する画像

---

## Step 1: GitHub リポジトリの準備

### 1-1. リポジトリを作成
1. GitHubにログイン
2. 右上の「+」ボタン → 「New repository」をクリック
3. リポジトリ名を入力（例：`konmariprompt`）
4. 「Public」を選択
5. 「Create repository」をクリック

### 1-2. ローカルにクローン
```bash
git clone https://github.com/あなたのユーザー名/リポジトリ名.git
cd リポジトリ名
```

---

## Step 2: Claude Code でサイトを作成

### 2-1. Claude Code に指示を出す

**基本的な指示の例：**
```
プロンプト集のサイトを作って。
- タイトルは「KONMARI prompt」
- カラーテーマはシアン、ピンク、イエロー
- プロンプトと画像を一覧表示
- コピーボタンを付けて
```

### 2-2. ファイル構成
Claude Code が以下のファイルを作成します：

```
プロジェクトフォルダ/
├── index.html      # メインのHTMLファイル
├── prompts.json    # プロンプトデータ
└── images/         # 画像フォルダ
    ├── image1.png
    ├── image2.png
    └── ...
```

### 2-3. 追加機能のリクエスト例

| やりたいこと | 指示の例 |
|------------|---------|
| デザイン変更 | 「もっとポップでおしゃれなデザインにして」 |
| 機能追加 | 「お気に入り機能を追加して」 |
| レイアウト変更 | 「スクロールなしで一画面に収まるようにして」 |
| 外部リンク | 「noteの記事へのリンクを追加して」 |

---

## Step 3: GitHub にプッシュ

### 3-1. Claude Code に指示
```
GitHubにプッシュして
```

### 3-2. 手動でプッシュする場合
```bash
git add -A
git commit -m "サイトを追加"
git push origin main
```

---

## Step 4: GitHub Pages を有効化

### 4-1. 設定画面を開く
1. GitHub のリポジトリページへ移動
2. 「Settings」タブをクリック
3. 左メニューの「Pages」をクリック

### 4-2. Pages を有効化
1. **Source** で「Deploy from a branch」を選択
2. **Branch** で「main」を選択
3. フォルダは「/ (root)」のまま
4. 「Save」をクリック

### 4-3. 公開URLを確認
数分後、以下の形式でサイトが公開されます：
```
https://ユーザー名.github.io/リポジトリ名/
```

---

## Step 5: サイトを更新する

### 5-1. 変更を依頼
```
カラーテーマを変更して
画像を追加して
新しいカテゴリーを追加して
```

### 5-2. プッシュ
```
GitHubにプッシュして
```

### 5-3. 反映を確認
- GitHub Pages の反映には1〜5分かかることがあります
- キャッシュが残っている場合は **Cmd + Shift + R**（Mac）または **Ctrl + Shift + R**（Windows）で強制リロード

---

## よくある問題と解決方法

### 問題1: 画像が表示されない
**原因**: 画像パスが間違っている or 画像ファイルがない

**解決方法**:
```
画像が表示されていないので確認して
```

### 問題2: サイトが更新されない
**原因**: ブラウザのキャッシュ

**解決方法**:
- 強制リロード: Cmd + Shift + R（Mac）/ Ctrl + Shift + R（Windows）
- シークレットウィンドウで開く
- URLに `?v=2` を追加（例：`https://...github.io/konmariprompt/?v=2`）

### 問題3: ローダーが消えない
**原因**: JavaScriptのエラー

**解決方法**:
```
開発者ツールでエラーが出ています。確認して修正して
```
（F12キーでコンソールのエラーメッセージを確認）

### 問題4: デザインがダサい
**解決方法**:
```
参考サイト: https://example.com
このテイストでデザインして
```
参考サイトのURLを添えて指示すると、そのテイストを参考にしてくれます。

## 実際に使った指示の例

### デザイン指示
```
トップがダサいので、もっとおしゃれに。
https://mearyworks.com/ このテイストで、
カラーイメージだけ今のままで作って
```

### レイアウト指示
```
プロンプトページはスクロールしなくてもいいように
一画面に画像とプロンプトとタイトルと詳細と
コピーボタンをレイアウトして欲しいです
```

### 機能追加指示
```
コピーだけじゃなくて、
お気に入り、シェア（Xに画像出す）も欲しい
```

### カテゴリー追加
```
noteサイトも入れて
https://note.com/konmari/n/n6f2c0fbffc6c
```

## Tips

### 画像の準備
- 画像は `images/` フォルダに入れる
- ファイル名は英数字とアンダースコアで（例：`snow_globe.png`）
- 日本語ファイル名は避ける

### 指示のコツ
1. **具体的に伝える** - 「おしゃれに」より「シアンとピンクのグラデーションで」
2. **参考サイトを添える** - イメージを共有しやすい
3. **スクリーンショットを見せる** - 問題箇所が明確になる
4. **一度に一つずつ** - 複数の変更は分けて依頼

### やり直したいとき
```
前のバージョンに戻して
```
または
```
この変更は取り消して
```
