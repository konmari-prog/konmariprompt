## 1. 今回のセッションでやったこと

- **春の新生活キャンペーンポスター用プロンプトの新規作成** — 完了（v10、まだ調整中）
  - 目的: スペシャルプレゼント用。画像+春の目標テキストを入力すると、プロの広告デザイン風の画像を生成するプロンプト
  - 公開用（prompts.json）ではなく、独立したプレゼント用プロンプトとして作成
  - ファイル: `spring_ad_design_prompt.md`
  - テスト対象AI: Gemini / Grok

- **10回のイテレーション** を経て以下の方向性に到達:
  - 入力: 人物画像 + 春の目標テキスト（必須）
  - 出力: 1:1正方形の春キャンペーンポスター画像
  - 人物: イラスト入力でも実写（フォトリアリスティック）に変換
  - 背景: リアル写真ではなくグラフィックデザイン背景（パステルグラデーション+ブロブ等）
  - タイトル: 目標テキストがメインビジュアル。VTuberロゴ風にモチーフを文字に織り込む
  - キャンペーン名: 「春の新生活目標応援キャンペーン」をリボン/バッジ等のグラフィック要素として配置
  - あしらい: 目標連動モチーフ + 春装飾。脇役として配置

## 2. うまくいったこと

- **既存の成功プロンプトの構造を踏襲する**アプローチが効いた。JSON構造の複雑なデザインシステムではなく、`**[AI指示]**` + `**[メインモチーフ]**` + `**[雰囲気]**` の3セクション構造にしたら出力が改善
- **AIに自由度を委ねる**方が良い結果が出た。「勝ちパターンをリサーチして適用」「AIが自由に最適化」という指示が効果的
- **参考画像（LINE LIVE, SHElikes風）を見せてもらった**ことで、求めるスタイルが「グラフィックデザイン背景 + ポップなタイポ」だと判明し、一気に方向性が定まった
- **カラーコード直書き禁止**にしたらGrokでのコード漏れが解消

## 3. うまくいかなかったこと・ハマったポイント

- **v1-v3: 「テキスト載せただけ」問題** — JSONで複雑なデザインシステムを指示しても、AIは結局イラストの上にテキストをフラットに配置するだけだった。原因: 画像生成AIはグラフィックデザインツールではない
- **v3: 枠がダサい問題** — 「帯やフレーム」と書くとAIがアンティーク調のコーナー装飾を生成した。「フレーム」というワード自体が原因
- **v5: カラーコード漏れ** — `#1A365D`をプロンプトに書くとGrokがそれを画像内テキストとして描画した。ブランド名「ルミネ」も漏れた
- **v9: あしらい過多** — 「ふんだんに盛り込む」と書いたらイラストが目標タイトルより目立ってしまった
- **桜の写真背景は「ポスター感」が出にくい** — AIの学習データ的に、グラフィック背景の方がキャンペーン広告っぽくなる

## 4. 主要な意思決定とその理由

| 意思決定 | 理由 |
|---------|------|
| 公開用(prompts.json)ではなくスペシャルプレゼント用 | ユーザーの要望。個別配布用 |
| 全面実写化（イラスト入力でも実写変換） | テスト結果でイラスト風はどうしても「テキスト載せただけ」感が消えなかった。実写の方がテキスト統合の学習データが多い |
| グラフィック背景（桜写真→パステルグラデ+ブロブ） | 参考画像（LINE LIVE/SHElikes風）を見せてもらい方向転換。グラフィック背景の方がポップでキャンペーン感が出る |
| VTuberロゴ風タイトルデザイン | モチーフを文字に織り込むことで、あしらいイラストに頼らずタイトル自体がビジュアル要素になる |
| JSONデザインシステム→成功プロンプトの3セクション構造 | 既存の成功プロンプト（お菓子、ドリンク等）がJSON使わずシンプルな構造で成功していたため |
| デザイン詳細をAIに委ねる | フォント、色、配置を全部指定すると「つまらなくなる」とのフィードバック。AIの創造性を活かす |

## 5. 学んだ教訓・注意点（Gotchas）

- **プロンプトにカラーコード(#1A365D等)を書くとGrokが画像内にそのまま描画する** → 色名（深いネイビー等）で記述
- **「フレーム」「帯」と書くとAIがダサい枠を出す** → 図形・カラー・背景で装飾と指示
- **実在ブランド名（ルミネ、Canva等）を参考として書くと画像に出る** → 全部削除
- **制約を増やすほどAIの出力はつまらなくなる** — 成功プロンプトは「何を作るか」は明確、「どう作るか」はAIに自由
- **画像生成AIが得意なフォーマット（雑誌表紙、商品パッケージ）は学習データが多いから成功しやすい** — 抽象的な「広告デザイン」は苦手
